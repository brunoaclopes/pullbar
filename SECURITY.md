# Security Policy

## Supported Versions

Security fixes are applied on the latest main branch and recent tagged release.

## Reporting a Vulnerability

Please do not open public issues for vulnerabilities.

- Preferred: use GitHub private vulnerability reporting for this repository.
- Include impact, reproduction steps, and any suggested mitigation.
- Redact sensitive data (tokens, hostnames, internal URLs).

We will acknowledge reports as quickly as possible and coordinate disclosure.
